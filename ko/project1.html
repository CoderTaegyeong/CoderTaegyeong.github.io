김태경의 포트폴리오

포트폴리오 웹사이트

2020년 6월 12일부터 작업 중.

깃헙에서 보기

목표

심플하고 현대적인 포트폴리오 만들기

사용 기술

  • 프론트엔드: HTML, CSS, JavaScript
  • 백엔드: PHP

기능들

  • 인덱스에서는 모든 페이지의 요약본을 볼 수 있어야 함.
  • 이력서에서는 사용자가 짧은 자기 소개, 학력, 기술역량을 볼 수 있도록 함.
  • 포트폴리오에서는 내가 넣은 프로젝트들을 한 눈에 볼 수 있고 하나씩 '자세히 보기' 버튼으로 설명을 읽을 수 있도록 함.
  • 연락하기에서는 연락 폼으로 사용자가 내게 이메일을 보낼 수 있고, 내가 그 메일을 받을 수 있어야 함.

실행 과정

  • 부트스트랩을 템플릿을 이용해 전체적인 디자인을 잡았다. 검은색 테마라 심플하면서도 세련된 느낌을 주는 Beckham 이라는 템플릿을 골랐다.
  • 불필요한 소셜 미디어 버튼들을 제거. 주소, 폰 번호, 지도 같은 개인 정보를 넣는 란은 삭제했다.
  • 이력서에서 짧은 자기 소개 밑에 세가지의 강점을 적고 학력 란에 독학한 기간도 적었다. 기술 역량을 설명하는 파트에서 애니메이션이 들어간 그래프들을 지우고 상세하게 내가 가진 기술적 역량을 솔직하게 썼다.
  • 간단한 PHP 코드로 하는 것보다 PHPMailer 라는 PHP 패키지를 이용해서 보내는 게 안전하므로 그 코드로 이메일을 보내도록 하였다. 패키지는 Composer 라는 패키지 매니저를 이용해 다운로드 받았다.
  • SMTP 서버 대부분이 유료이기 때문에 무료인 구글의 SMTP를 이용했다. 구글 개발자 콘솔 페이지에서 PHPMailer 라는 이름의 프로젝트를 만들고 Gmail API를 켰다. 새 클라이언트 ID를 만들고 리다이렉트 URI에 http://localhost/phpmailer/get_oauth_token.php 을 넣어줬다. get_oauth_token.php에 내 클라이언트 ID와 시크릿 키를 넣고 그 링크를 실행해서 리프레시 토큰을 가져오고 emailer.php에다 넣었다. PHPMailer OAuth2 코드에 적용했다. 이 방식을 굳이 선택한 이유는 내 구글 계정 정보를 코드에 직접 쓰지 않아도 되기 때문이었다.
  • 연락폼을 테스트 하기 위해 XAMPP 로컬호스트 서버를 켜고 해 보았다. 메일은 성공적으로 발신, 수신이 되지만 보내는 사람 이메일이 내가 넣은 지메일 주소로만 나왔다. "AddReplyTo()" 함수를 넣어 "setFrom()"을 대신해 보내는 사람의 이메일 주소를 받아오도록 함.
  • HTML 페이지들을 "ko" 폴더에 따로 복사하여 한국어 페이지를 작업했다. 인덱스 페이지를 작업하면서 langmenu 라이브러리를 이용해서 언어 선택 버튼을 왼쪽 상단에 고정했다.
